# 重力四子棋游戏策划和需求文档

## 1. 游戏概述

### 1.1 游戏名称
重力四子棋

### 1.2 游戏类型
策略类桌面游戏

### 1.3 游戏平台
Web端（PC/移动设备）

### 1.4 游戏简介
重力四子棋是一款基于传统四子棋规则的创新策略游戏，采用垂直立体结构，棋子下落时自动填充最底层空位。玩家需要在横、竖、斜任意方向形成四子连线以获得胜利，同时需要预判对手策略并封锁进攻路线。

## 2. 核心玩法

### 2.1 游戏规则
- **棋盘结构**：采用垂直立体结构，默认使用5×5×5的三维空间（可根据实际情况调整为更适合网页展示的二维模拟三维效果）
- **棋子放置**：玩家选择列后，棋子会自动下落并填充该列的最底层空位
- **胜利条件**：率先在横、竖、斜任意方向形成四子连线
- **游戏流程**：
  1. 玩家进入网站首页，默认创建一个房间
  2. 房间满2人后游戏开始
  3. 玩家轮流放置棋子
  4. 检测是否有玩家达成胜利条件
  5. 游戏结束，显示胜利方或平局

### 2.2 游戏特色
- 无需注册登录，进入首页后默认创建房间，右上角也可以输入房间号即可加入其他房间的游戏
- 支持观战模式，当房间已满时可进入观战
- 实时显示游戏状态和玩家信息
- 简洁直观的界面设计

## 3. 系统架构

### 3.1 技术栈
- **后端**：Python + FastAPI
- **前端**：uni-app
- **通信**：WebSocket（实时游戏状态同步）
- **存储**：内存存储（无需数据库）

### 3.2 系统模块

#### 3.2.1 后端模块
- **房间管理**：创建、销毁、加入、管理游戏房间
- **游戏逻辑**：处理棋子放置、胜负判断、游戏状态更新
- **WebSocket服务**：实时推送游戏状态和消息
- **观战系统**：处理观战请求和状态同步

#### 3.2.2 前端模块
- **房间界面**：显示房间号、玩家信息、棋盘
- **棋盘交互**：处理玩家棋子放置操作
- **游戏状态展示**：实时显示游戏进展和结果
- **观战模式**：无操作权限的观战界面

## 4. 功能需求

### 4.1 核心功能
1. **房间系统**
   - 自动生成房间号
   - 输入房间号加入游戏
   - 房间人数限制（最多2人游戏，支持多人观战）
   - 房间状态管理（等待中、游戏中、已结束）

2. **游戏逻辑**
   - 棋子放置与重力下落效果
   - 胜负判断（横、竖、斜四子连线）
   - 游戏回合管理
   - 游戏状态同步

3. **玩家系统**
   - 无需注册登录，临时玩家身份
   - 玩家信息显示（颜色、状态）
   - 轮到谁下棋的提示

4. **观战系统**
   - 房间已满时自动进入观战模式
   - 实时观看游戏过程
   - 无法进行游戏操作

### 4.2 界面需求
注意只需要开发一个页面：
1. **页面总体框架（首页）**
   - 中央显示棋盘(游戏区域)
   - 中央顶部显示房间号
   - 左右两边显示玩家信息
   - 右上角显示"加入房间"输入框

1.2. **游戏区域**
   - 二维棋盘可视化（模拟三维效果）
   - 棋子放置交互区域
   - 游戏状态提示
   - 胜负结果显示

1.3. **观战界面**
   - 直接显示游戏界面，但无操作权限
   - 显示观战状态

## 8. 技术挑战与解决方案

### 8.1 三维棋盘的可视化
- **挑战**：在二维网页上展示三维棋盘
- **解决方案**：使用等轴测投影或分层展示的方式，模拟三维效果

### 8.2 实时通信
- **挑战**：确保游戏状态实时同步
- **解决方案**：使用WebSocket实现低延迟通信

### 8.3 胜负判断算法
- **挑战**：在三维空间中高效判断四子连线
- **解决方案**：预计算所有可能的连线方向，每次落子后检查相关方向

### 8.4 多设备适配
- **挑战**：确保在不同设备上的良好体验
- **解决方案**：使用响应式设计和自适应布局

## 9. 总结

重力四子棋是一款融合传统四子棋规则与立体空间思维的创新策略游戏。通过垂直立体结构和重力下落机制，为玩家提供了全新的游戏体验。本项目将使用Python+FastAPI后端和uni-app前端，实现一个无需注册登录、支持实时对战和观战的网页版重力四子棋游戏。

游戏的核心价值在于锻炼玩家的空间思维能力和策略规划能力，同时提供了简单易用的界面和流畅的游戏体验。通过本项目的开发，玩家可以在任何有网络的设备上随时随地享受重力四子棋的乐趣。
